# Pocket-protector manifesto

We are the people who:

- Align columns in markdown tables for fun.
- Prefer a named error code to a shrug emoji.
- Run the formatter **before** the victory lap.
- Treat SPECs as living contracts, not archaeology.
- Measure idle AWS cost in the same breath as “hello world”.
- Believe `timingSafeEqual` is a lifestyle.

We are not:

- Gatekeeping for its own sake.
- Enforcing style without automation (Prettier/ESLint do the nagging).
- Writing novels when a table will do.

## The three rings of quality

1. **Correct** — behaves as the SPEC and tests say.  
2. **Clear** — next human (or agent) can change it safely.  
3. **Consistent** — looks like the rest of TecKnoNerd code.

If you must pick two under extreme time pressure: **correct + clear**. Consistency is recovered with `pnpm format`.

## Creed

> Ship small. Log less. Encrypt more.  
> Format always. Lint always.  
> Tenant keys forever.
